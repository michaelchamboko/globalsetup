# Architecture Reviewer Profile

Review changes for design pattern alignment, modular separation, and system interface boundaries.

## Checklist

- [ ] **Modular Separation**: Component and logic boundaries are respected. No circular dependencies.
- [ ] **Pattern Consistency**: New files follow the repository's structural conventions (e.g. MVC, Clean Architecture).
- [ ] **Interface Design**: Public methods, props, and APIs have clean, stable interfaces.
- [ ] **State Management**: Local, component, and global state boundaries are clearly defined.
- [ ] **Extensibility**: Code is structured to allow future enhancements without breaking core logic.
