# Documentation Reviewer Profile

Review changes for documentation accuracy, completeness, and clarity.

## Checklist

- [ ] **Accuracy**: Code signatures, endpoints, and file paths mentioned in documentation match the codebase.
- [ ] **Readme Updates**: Project README and guide documents are updated to reflect new features or configuration steps.
- [ ] **Inline Documentation**: Module boundaries and complex algorithms have explaining comments focusing on *why*, not *what*.
- [ ] **API Reference**: Swagger/OpenAPI specifications or type definitions are kept in sync with API routes.
- [ ] **Configuration Guide**: New environment variables or setup prerequisites are documented.
