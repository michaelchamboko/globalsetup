# Build Plan Templates

Build plans make Step 2 visible before implementation begins. They sit between contracts and module plans:

```text
PRD -> contracts -> build plans -> module plans -> task cards -> intended-location validation
```

Use these templates to create:

- `build-pack/build-plans/01-build-plan-index.md`
- `build-pack/build-plans/02-ui-ux-build-plan.md`

User-facing products must include the UI/UX build plan.
